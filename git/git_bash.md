## git bash

### git bash 기초

- 폴더 만들고 우클릭 → git bash here 클릭하면 git bash가 해당 디렉토리로 열린다.

  

### git bash 리눅스 명령어

* git bash는 리눅스 기반으로 git bash를 사용하려면 리눅스 기초 명령어에 대해 알아야 한다.

  

* `~` 는 홈버튼을 의미한다. (보통 `~`를 `홈`이라고 부른다)

  * 일반적인 경우 `c\users\계정이름` 경로가 홈으로 설정되어 있다.
  * `cd ~` : 한번에 홈으로 이동하는 방법


  

* `/`는 최상위 경로를 의미한다. (보통 `/`를 `루트`라고 부른다)

  * `cd /` :  한번에 최상위 경로로 이동하는 방법



* 리눅스에서 `.`은 현재폴더(+하위폴더 모두 포함)를 가리킨다.



* `pwd` (print working directory): 현재 작업중인 디렉토리를 보여준다.

  

* `cd` (change directory): 다른 경로로 이동한다.

  

* `cd..` : 현재 경로에서 한 단계 상위 폴더로 이동한다.

  

* `dir` : 현재 폴더의 내용물들을 보여준다.

  

* `ls -a`

  * 현재 폴더의 모든 내용을 보여달라는 것(a는 all의 의미)

  * 입력시 ./ ../ 출력됨 → .은 현재 폴더, ..은 상위폴더를 의미

  * .으로 시작하는 폴더는 숨김처리되어 있음을 

    

* `touch test.py` : test.py라는 파일이 생성된다. 

  * `touch a.txt b.txt c.txt `
    → `touch` 명령어로 여러개 파일 한번에 생성 가능
    → 빈칸하나를 기준으로 여러개 파일 생성가능

    

* `mkdir test`: test라는 폴더가 생성된다  (`mkdir`: **m**a**k**e  **dir**ectory)

  

* `code .` 

  *  `code`는 VS code를 실행하는 명령어  
  * `. `은 현재 폴더를 기준으로 실행하라는 것



* `rmdir <폴더명>` : 폴더 삭제 



* `rm <파일명>` : 파일삭제

### git bash 관련 기타 팁



* 자동완성기능 (`tab키`)
  * 첫글자를 입력하고 `tab키`을 누르면 해당 폴더에서 그 첫글자로 시작하는 대상으로 자동완성해준다.
  * 그 상태에서 `tab키`을 한 번 더 눌러주면 그 첫글자로 시작하는 모든 대상을 보여준다. 
  * 그리고 글자를 입력하다 `tab키`을 눌렀을 때, 해당 부분으로 시작하는 파일이 **하나**라면 자동완성해준다.



* git bash에서 붙여넣기 하는법
  → `shift + insert 키`
